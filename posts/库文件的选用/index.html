库文件的选用

May 8, 2021 2-minute read

本次项目我们选用了控制器为SH1106的1.3英寸OLED屏作为显示输出,但Adafruit官方并没有提供SH1106的驱动程序.

查阅相关资料发现对于SH1106,有两个可供选择的库,分别是u8g2和基于adafruit_SSD1306修改的第三方库.

Adafruit_SH1106 vs u8g2

Adafruit_SH1106

以下是该库的Readme描述

Adafruit graphic library for SH1106 driver lcds.
some small oled lcd use SH1106 driver.
I change the adafruit SSD1306 to SH1106
SH1106 driver similar to SSD1306. thus, just change the display() method.
However, SH1106 driver don’t provide several functions such as scroll commands.
Adafruit-GFX-Library https://github.com/adafruit/Adafruit-GFX-Library

关于sh1106,有一点需要提及。

官方Adafruit_GFX库选择在堆区域中分配图像缓存，而sh1106的作者Wonho-maker将其修改为全局储存区，即arduino内存概述中提到的静态数据(Static data)区域。

我们可以通过下方的程序来比较这两者之间的区别

/* 
   Adafruit SSD1306和SH1106之间的内存消耗比较(byte)
   SSD1306 sketch=13082, global= 362, local=1686
   SH1106  sketch=11190, global=1375, local= 673
*/
 
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>     // SSD1306
//#include <Adafruit_SH1106.h>    // SH1106
 
#define OLED_RESET 4
Adafruit_SSD1306 oled(128, 64, &Wire, OLED_RESET);   // SSD1306
//Adafruit_SH1106 oled(OLED_RESET);   // SH1106
 
void setup()   {          
  oled.begin(SSD1306_SWITCHCAPVCC, 0x3C); // SSD1306 select 3C or 3D (set your OLED I2C address)
  //oled.begin(SH1106_SWITCHCAPVCC, 0x3C);  // SH1106
}
 
void loop() {                // Hello, world!
  oled.clearDisplay();
  oled.setTextSize(1);
  oled.setTextColor(WHITE);
  oled.setCursor(0, 0);
  oled.println("Hello, world!");
  oled.display();
  delay(2000);
}

对于同一段程序，这两库的内存消耗分别为：

sh1106

ssd1306

sketch global local
SSD1306 13082 362 1686
SH1106 11190 1375 673

为了更详细地说明，SSD1306的本地内存区域应该是1686字节，但是减去屏幕缓冲区(1024字节)后，剩余的就是662字节，与SH1106几乎相同，或者说可用空间更少

换句话说，两者之间的内存大小差异是在global还是local中分配屏幕缓冲区，其实并没有实质性差异

但由于这一点,使用Adafruit_SH1106时，更容易得到编译警告(当local内存区域低于约500字节以下时警告)

但其实内存使用量几乎与SSD1306相同，因此不必担心太多.

此外Adafruit_sh1106似乎将I2C时钟设置为100kHz。为了使其达到400kHz，还需要进行额外的校正

u8g2

U8g2 is a monochrome graphics library for embedded devices.

The Arduino library U8g2 can be installed from the library manager of the Arduino IDE. U8g2 also includes U8x8 library. Features for U8g2 and U8x8 are:

U8g2

  • Includes all graphics procedures (line/box/circle draw).
  • Supports many fonts. (Almost) no restriction on the font height.
  • Requires some memory in the microcontroller to render the display.

U8x8

  • Text output only (character) device.
  • Only fonts allowed with fixed size per character (8x8 pixel).
  • Writes directly to the display. No buffer in the microcontroller required.

比较

由于Adafruit_GFX使用完整缓冲区方法，因此它使用的内存比采用页面缓冲区方法的u8g2多

但是，由于整个屏幕的信息都存储在缓冲器中，因此取决于显示条件，可以进行高速处理

换句话说，两者都有优点和缺点，所以我认为不能说哪个更好

目前来说，我们更倾向于选择基于adafruit_SSD1306的第三方库

或许后期我们可以比对官方库和其修改的部分，作为参考进行二次修改
